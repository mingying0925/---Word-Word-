## **1. 产品背景与目标**

- **痛点**：手动填表易错、格式混乱，教师校对成本高。
- **目标**：教师上传任何带 **Word 书签** 的申报表模板，系统自动生成在线填报页面，学生填写后导出与原始表格样式完全一致的 Word 文件。
- **V1.0 范围**：仅实现单个教师、单个活动的完整流程。暂不涉及多活动管理、批量导出、异步任务、图片上传等。
- **V2.0 范围**：在 V1.1 双入口架构基础上，补齐安全短板、真实认证、图片填充、活动生命周期管理，并完成生产部署改造。

## **2. 系统架构 (混合语言)**

```
[ 学生/教师 浏览器 ]
        |
        | HttpOnly Cookie (JWT, SameSite=Lax)
        v
[ Java Spring Boot 主服务 (localhost:8080) ]  ←→  [ Python Flask 导出微服务 (localhost:5000) ]
        |                                              |
        | CSRF Filter → RateLimit Filter → TraceId Filter → JwtInterceptor
        |                                              |
[ MySQL 8.0 (生产) / H2 (开发) ]              [ python-docx 处理 Word ]
        |                                              |
[ Flyway 管理 schema 迁移 ]                   [ 127.0.0.1 仅内网调用 ]
```

- **Java 主服务**：负责业务逻辑、数据库操作、页面渲染、用户交互、JWT 鉴权、CSRF 防护、限流、traceId 追踪。
- **Python 微服务**：仅负责 Word 书签解析与数据填充，监听 127.0.0.1，仅 Java 主服务调用。
- **安全链路**：TraceIdFilter → CsrfFilter → RateLimitFilter → JwtInterceptor → Controller。
- **数据加密**：身份证号 AES-256-GCM 加密存储（IdCardConverter 自动加解密）。

### V1.1 架构变更
- 新增 `IndexController` 根路径门户页（角色选择）。
- 新增 `JwtInterceptor` 基于 URL 前缀的角色鉴权（`/teacher/**` vs `/student/**`）。
- 新增 `JwtUtil` 无状态 Token 生成/解析（HS256，HttpOnly Cookie 传输）。
- 教师端与学生端登录流程完全分离，独立校验规则。

## **4. 功能需求 (MVP V1.0 已交付)**

### **4.1 教师端**

#### **F1. 上传模板并解析书签** ✅
- 教师进入"创建活动"页面，上传一份 **已插入书签** 的 `.docx` 模板。
- 系统自动调用 Python 微服务解析书签。
- 解析结果返回 **书签名称、所在表格坐标（行/列/合并信息）**。
- 前端展示解析出的字段列表。

#### **F2. 生成活动并预览表单** ✅
- 教师确认字段后保存活动（活动包含模板文件路径、解析出的书签坐标 JSON）。
- 可点击"预览表单"查看学生端将看到的仿原表在线表格。

#### **F3. 查看报名记录与导出 Word** ✅
- 在活动详情中列出所有学生提交的报名数据。
- 每条记录旁有 **"导出 Word"** 按钮，点击后由 Java 调用 Python 将数据填入原模板，返回下载链接。
- 支持批量导出 ZIP（V1.1 增补）。

### **4.2 学生端**

#### **F4. 身份验证与唯一提交** ✅
- 学生访问活动链接，输入 **学号 + 身份证号**。
- 系统校验此组合是否已提交过。若未提交，允许进入填表。
- 身份证号采用 GB 11643-1999 校验码算法验证（V1.1 增补）。
- 一旦提交，不可重复提交。

#### **F5. 在线填报表单 (仿原表结构)** ✅
- 系统根据教师模板的 **书签坐标**，动态渲染一个 HTML 表格。
- 表格结构与原 Word 模板完全一致（合并单元格、行列布局相同）。
- 有书签的单元格变为 **可编辑的输入框** 或 **下拉/单选框**（根据书签名称智能推断，如"性别"对应单选框，其余默认文本框）。
- 非书签单元格仅展示原始文字（只读）。
- 学生填写完毕后点击"提交"，数据以 JSON 格式保存到数据库。

## **5. 非功能性需求 (MVP 简化)**

- 数据校验：学号、身份证号非空；身份证号 GB 11643-1999 校验；文本输入防 XSS（Thymeleaf 自动转义）。
- 安全性：身份证号暂明文存储（MVP 阶段），V2.0 迭代加密。
- 兼容性：表格在移动端可滚动，不强制完美适配。

## **10. 前端动态表单生成逻辑 (Thymeleaf)** ✅

- Controller 层解析 `bookmarks_json`，预计算每个单元格的 `rowspan` / `colspan` 和控件类型。
- 封装为 `List<TableData>` 传入模板。
- 模板遍历表格结构，渲染 `<table>`：
  - 若单元格包含书签：
    - `type == "radio"`：渲染 `<input type="radio">` 组合
    - `type == "date"`：渲染 `<input type="date">`
    - `type == "image"`：渲染文件上传占位（V1 仅占位，V2 实现真实上传）
    - 其他：渲染 `<input type="text" name="书签名">`
  - 否则：直接输出文本（只读）。
- 提交时序列化表单为 JSON：`{"书签名": "值", ...}`

## **12. V2.0 迭代规划 - 安全加固（P0 优先级）** ✅ 已交付

### **12.1 真实教师认证体系** ✅
**现状**：已实现完整的教师认证体系。

**目标**：
- 新增 `teachers` 表，存储工号、BCrypt 密码哈希、姓名、状态。
- 教师登录改为数据库校验：`teacherId + password` → BCrypt 比对。
- 提供初始化脚本/CLI 命令创建首个教师账号。
- 密码强度策略：≥8 位，含字母+数字。

**验收**：
- 错误密码登录失败并提示。
- 密码以 BCrypt 哈希存储，数据库泄露不可逆推。
- 首次部署可通过命令行创建管理员账号。

### **12.2 JWT 密钥外部化** 🔲
**现状**：`jwt.secret` 硬编码在 `application.yml`。

**目标**：
- 密钥改为环境变量 `${JWT_SECRET}` 注入，启动时校验非空且 ≥32 字节。
- 提供 `application-dev.yml` / `application-prod.yml` 多环境配置。
- `.env` / `application-local.yml` 加入 `.gitignore`。

### **12.3 Cookie 安全属性补全** 🔲
**现状**：Cookie 仅设置 `HttpOnly`，缺少 `Secure` 与 `SameSite`。

**目标**：
- 生产环境 Cookie 增加 `Secure`（仅 HTTPS 传输）。
- 增加 `SameSite=Lax`（防 CSRF）。
- 通过 `response.setHeader("Set-Cookie", ...)` 手动拼接（Servlet API 原生不支持 SameSite）。

### **12.4 异常信息脱敏** 🔲
**现状**：`GlobalExceptionHandler` 将 `"系统异常：" + e.getMessage()` 直接展示给用户，可能泄露 SQL/堆栈。

**目标**：
- 业务异常展示用户友好消息。
- 系统异常统一展示"系统繁忙，请稍后重试"，详细堆栈仅写入日志。
- 日志中记录完整异常链（含 traceId）。

### **12.5 身份证号加密存储** 🔲
**现状**：`submissions.id_card` 明文存储。

**目标**：
- 采用 AES-256-GCM 对称加密，密钥由环境变量注入。
- 数据库仅存密文，查询时解密。
- 展示时脱敏（保留前 6 后 4 位）。
- 唯一性校验改为密文比对（同密钥同明文 → 同密文）。

### **12.6 生产环境配置隔离** 🔲
**现状**：H2 控制台暴露、`ddl-auto: update`、`show-sql: true`。

**目标**：
- `application-prod.yml`：关闭 H2 控制台、`ddl-auto: validate`、关闭 `show-sql`。
- 引入 Flyway 管理 schema 迁移。
- 切换至 MySQL 8.0+。

## **13. V2.0 迭代规划 - 功能补齐（P1 优先级）**

### **13.1 图片上传与填充** ✅ (V1.2 已交付)
**现状**：已实现完整的证件照上传与 Word 填充闭环。

**已交付功能**：
- 学生端表单中 `type == "image"` 的书签渲染为带预览的文件上传控件（支持点击选择与拖拽上传）。
- 前端实时预览所选图片，校验文件类型（JPG/PNG/GIF）与大小（≤2MB），超限给出友好提示。
- 后端 `StudentController.handleSubmit` 接收图片文件，保存至 `uploads/images/{activityId}_{studentId}/`，绝对路径存入 `formDataJson`。
- Python 微服务 `fill_docx` 自动检测值为图片文件路径的字段，调用 `run.add_picture` 将图片插入对应书签单元格，宽度自适应（1.5 英寸）。
- `insert_bookmarks.py` 脚本已内置 `照片` 关键词识别，自动插入 `photo` 书签。

**技术细节**：
- 图片路径以绝对路径存储在 `form_data_json` 中，Java 与 Python 需部署在同一机器或共享文件系统。
- 导出时 Python 端通过 `os.path.isfile()` + 扩展名校验判断是否为图片，非图片字段按文本填充。

### **13.2 活动生命周期管理** 🔲
**现状**：活动创建后无法截止或删除（`deleteActivity` 方法存在但无 UI 入口）。

**目标**：
- 活动列表页增加"截止报名"按钮（status 0→1）。
- 增加"删除活动"按钮（二次确认，级联删除 submissions + 模板文件）。
- 截止后学生端登录页提示"该活动已截止报名"。
- 截止后教师端仍可查看与导出，但不可再提交。

### **13.3 截止时间前端展示与拦截** 🔲
**现状**：后端 `submitRegistration` 已校验截止时间，但前端无展示与拦截。

**目标**：
- 学生登录页与填表页展示活动截止时间。
- 截止时间到达后，填表页"提交"按钮禁用并提示。
- 后端保持原有校验作为兜底。

### **13.4 教师活动管理增强** 🔲
**现状**：无活动搜索、筛选、分页。

**目标**：
- 活动列表支持按状态（报名中/已截止）筛选。
- 支持按名称模糊搜索。
- 分页（每页 20 条）。
