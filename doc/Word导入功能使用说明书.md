## 一、功能概述

SkillBridge 是一套「职业技能等级认定报名与文档自动化系统」。其核心能力是：

```
教师上传 Word 模板 → 系统自动解析书签 → 生成仿原表在线表单
→ 学生在线填写（含证件照上传） → 一键导出与原模板样式一致的 Word 文件
```

**「导入 Word」** 指的是教师上传一份 **已插入书签的 `.docx` 模板**，系统解析其中的表格结构与书签位置，自动生成在线填报页面。这是整个流程的起点，模板质量直接决定后续表单与导出效果。

## 二、环境准备

在使用本功能前，请确认以下两个服务已启动：

| 服务 | 开发环境地址 | 生产环境地址 | 启动方式 |
|------|-------------|-------------|---------|
| Java 主服务 | http://localhost:8080 | http://<服务器IP>:8088 | `cd skillbridge-web && mvn spring-boot:run` |
| Python 导出微服务 | http://localhost:5000 | http://127.0.0.1:5000 | `cd skillbridge-export && python app.py` |

> **说明**：生产环境 Python 微服务仅监听 127.0.0.1，不对外暴露，由 Java 主服务内部调用。

**验证方法**：
- 浏览器访问 Java 主服务地址，能看到角色选择门户页。
- 浏览器访问 `http://localhost:5000/api/health`（开发）或 `http://127.0.0.1:5000/api/health`（生产），返回 `{"status":"ok"}` 即正常。

> **注意**：Python 微服务负责解析书签和填充 Word，如果未启动，上传模板时会报错。

## 三、前置准备：制作带书签的 Word 模板

这是最关键的一步。系统不会识别任意 Word 文件，它只识别 **Word 书签（Bookmark）**。

### 3.1 什么是书签

书签是 Word 原生的定位标记，用于标记文档中某个位置或区域。SkillBridge 通过书签定位「需要学生填写的单元格」。

### 3.2 如何插入书签（手动方式）

1. 用 Microsoft Word 打开你的申报表模板（`.docx`）。
2. 选中需要学生填写的单元格（例如「姓名」右侧的空格）。
3. 点击菜单栏 **插入 → 书签**（或快捷键 `Ctrl+Shift+F5`）。
4. 在弹出的对话框中输入书签名称（如 `name`），点击「添加」。
5. 对所有需要填写的单元格重复上述操作。

### 3.3 书签命名规范（重要）

书签名称决定了学生端表单的控件类型和导出文件的命名。系统会根据书签名自动推断控件类型：

| 书签名包含关键词 | 推断出的控件类型 | 示例书签名 |
|------------------|-----------------|-----------|
| `姓名` / `name` / `xm` | 文本输入框 | `name`、`姓名`、`xm` |
| `性别` | 单选框（男/女） | `gender`、`性别` |
| `日期` / `时间` | 日期选择器 | `birth_date`、`申报日期` |
| `照片` / `photo` | 图片上传控件 | `photo`、`照片` |
| 其他 | 普通文本输入框 | `phone`、`id_number` |

> **导出文件命名依赖**：系统会从学生填写的数据中查找键名包含 `姓名`、`name`、`xm` 的字段值，作为导出 Word 的文件名。因此 **至少要有一个书签的名称包含上述关键词**，否则导出文件将命名为 `未命名申报表.docx`。

### 3.4 推荐的书签命名清单

针对「广东省职业技能等级认定个人申报表」，项目内置了书签插入脚本，推荐使用以下书签名：

| 字段 | 书签名 | 字段 | 书签名 |
|------|--------|------|--------|
| 姓名 | `name` | 申报职业 | `declare_occupation` |
| 性别 | `gender` | 申报等级 | `declare_level` |
| 出生年月 | `birth_date` | 考试类型 | `exam_type` |
| 证件类型 | `id_type` | 申报条件 | `declare_condition` |
| 证件号码 | `id_number` | 工作单位 | `work_unit` |
| 手机号码 | `phone` | 现从事岗位 | `current_position` |
| 考生类别 | `candidate_type` | 当前最高学历 | `highest_education` |
| 学历专业 | `education_major` | 发证时间 | `education_date` |
| **证件照** | **`photo`** | | |

工作经历区域按行编号：`work_start_1`、`work_company_1`、`work_position_1`、`work_contact_1` ...（1~5 行）。

> **证件照书签**：在模板中包含「照片」文字的单元格插入名为 `photo` 的书签后，学生端会显示图片上传控件，导出的 Word 中会自动插入真实证件照。

### 3.5 使用脚本自动插入书签（推荐）

项目提供了自动化书签插入脚本 `skillbridge-export/insert_bookmarks.py`，可自动识别关键词并插入书签：

```bash
cd D:\development\SkillBridge\skillbridge-export
python insert_bookmarks.py
```

**使用前提**：
1. 将原始 `.doc` 模板文件放在 `skillbridge-export/` 目录下，文件名需包含「申报表」三个字。
2. 已安装 `pywin32`（用于 `.doc` → `.docx` 转换，仅 Windows 支持）。
3. 已安装 `python-docx` 和 `requests`。

**脚本执行流程**：
1. 自动查找目录下的 `.doc` 申报表文件。
2. 调用 Word COM 组件转换为临时 `.docx`。
3. 按预定义的关键词清单（姓名、性别、出生年月、照片等）在表格中定位单元格。
4. 在目标单元格插入书签（关键词右侧或同单元格）。
5. 保存为 `2.广东省职业技能等级认定个人申报表_带书签_final.docx`。
6. 自动调用解析接口校验书签是否插入成功。

> **输出示例**：
> ```
> + 书签 "name"（关键词 "姓名"）
> + 书签 "gender"（关键词 "性别"）
> + 书签 "birth_date"（关键词 "出生年月"）
> + 书签 "photo"（关键词 "照片"）
> ...
> 书签插入完成，共插入 32 个书签
> ```

### 3.6 手动验证书签

插入书签后，可在 Word 中验证：
- **方法 1**：打开文件，按 `Ctrl+G`（定位）→ 选择「书签」→ 能看到书签列表。
- **方法 2**：文件 → 选项 → 高级 → 勾选「显示书签」，文档中书签位置会显示方括号标记。

## 四、教师端操作流程：上传模板与创建活动

### 步骤 1：登录教师端

1. 浏览器访问系统地址（开发环境：`http://localhost:8080`；生产环境：`http://<服务器IP>:8088`）。
2. 在门户页点击「教师入口」。
3. 输入教工号和密码，点击登录。
   - 开发环境默认账号：`admin` / `admin123`
   - 生产环境账号：由管理员分配（首次部署密码通过环境变量 `TEACHER_DEFAULT_PASSWORD` 设置，建议尽快修改）

### 步骤 2：进入创建活动页面

1. 登录后自动跳转到「活动列表」页面。
2. 点击「创建活动」按钮。

### 步骤 3：填写活动信息并上传模板

在创建活动页面填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 活动名称 | 必填，标识本次报名活动 | `2026 年职业技能等级认定报名` |
| 报名截止时间 | 选填，留空表示不限制 | `2026-07-31 23:59` |
| Word 模板文件 | 必填，仅支持 `.docx` 格式 | `申报表_带书签.docx` |

操作：
1. 点击「选择文件」按钮，选择已插入书签的 `.docx` 模板。
2. 点击「创建活动」按钮。

### 步骤 4：确认解析结果

上传后系统自动跳转到「确认字段」页面，展示：

- **活动名称** 和 **截止时间**。
- **书签字段数**：解析到的书签总数。
- **书签字段列表**：每个书签的名称、所在表格、行号、列号、控件类型。
- **表格结构预览**：仿原表的 HTML 表格预览，书签单元格高亮显示。

**请仔细核对**：
1. 书签数量是否符合预期？如果为 0，说明模板中没有书签，需重新制作模板。
2. 控件类型是否正确？例如「性别」应显示为 `radio`，「照片」应显示为 `image`。
3. 表格结构是否与原 Word 一致？

- 确认无误 → 点击「确认保存」。
- 发现有误 → 点击「取消」，重新制作模板后再次上传。

### 步骤 5：活动创建成功

保存成功后跳转回活动列表，显示成功提示「活动创建成功：xxx」。此时活动状态为「报名中」，学生可以开始填报。

## 五、学生端操作流程：在线填报

### 步骤 1：学生登录

1. 学生访问系统地址（开发环境：`http://localhost:8080`；生产环境：`http://<服务器IP>:8088`），点击「学生入口」。
2. 输入 **学号** 和 **身份证号**，选择要报名的活动。
3. 系统校验：
   - 身份证号格式（GB 11643-1999 校验码算法）。
   - 该学号+身份证组合是否已提交过（不可重复提交）。
   - 活动是否已截止。

### 步骤 2：填写表单

- 表单页面会渲染出与 Word 模板完全一致的表格结构。
- 有书签的单元格变为可编辑控件（文本框/单选框/日期选择器/图片上传）。
- 非书签单元格为只读文字。
- 填写完毕点击「提交」。

### 步骤 2.1：上传证件照（如模板含照片书签）

如果模板中包含 `photo` 或 `照片` 书签，表单对应位置会显示一个 **证件照上传区域**：

1. **点击上传**：点击虚线框区域，选择本地图片文件。
2. **拖拽上传**：直接将图片文件拖拽到虚线框区域。
3. **实时预览**：选择图片后立即显示预览缩略图，确认无误后再提交。
4. **重新选择**：如需更换，再次点击区域选择新图片即可覆盖。

**证件照要求**：

| 项目 | 限制 |
|------|------|
| 格式 | JPG / PNG / GIF |
| 大小 | ≤ 2MB（前后端双重校验，超限会提示） |
| 建议尺寸 | 295×413 像素（标准证件照） |

> **注意**：如果未上传证件照，导出的 Word 中照片位置将为空白，不影响其他字段的填写与导出。

### 步骤 3：提交成功

提交后跳转到成功页面，数据以 JSON 格式存入数据库，证件照文件保存至服务器 `uploads/images/{活动ID}_{学号}/` 目录。同一学号+身份证不可重复提交。

## 六、教师端操作流程：导出 Word

### 6.1 单个导出

1. 在活动列表点击某活动进入「提交列表」页面。
2. 每条学生记录右侧有「导出 Word」按钮。
3. 点击后浏览器自动下载文件。

**文件命名规则**：
- 从学生填写的表单数据中查找姓名（键名包含 `姓名`、`name`、`xm`，按优先级匹配）。
- 文件名格式：`{姓名}.docx`（例如 `张三.docx`）。
- 如果找不到姓名字段，文件名为 `未命名申报表.docx`。

### 6.2 批量导出 ZIP

1. 在提交列表页面点击右上角「一键批量导出所有 Word」按钮。
2. 浏览器下载一个 ZIP 压缩包。

**ZIP 内文件命名规则**：
- 压缩包名：`【{活动名称}】全员申报表汇总.zip`。
- 内部每个 Word 文件名：`{姓名}.docx`（与单个导出规则一致）。
- 如果有重名，自动追加序号：`张三.docx`、`张三(1).docx`。
- 如果某学生导出失败，会生成一个 `{姓名}_导出失败.txt` 占位文件说明原因，不影响其他学生的导出。

### 6.3 导出文件说明

- 导出的 Word 文件 **与原始模板样式完全一致**（表格结构、合并单元格、字体格式均保留）。
- 学生填写的数据被精准填入对应书签位置。
- 图片类书签（如照片）会以 1.5 英寸宽度插入到对应单元格。

## 七、活动管理

### 7.1 截止活动

- 在活动列表点击「截止报名」按钮，活动状态变为「已截止」。
- 截止后学生无法再提交，但教师仍可查看与导出。

### 7.2 删除活动

- 在活动列表点击「删除」按钮（需二次确认）。
- 删除后会级联删除该活动下所有学生提交记录和模板文件，**操作不可恢复**。

## 八、常见问题与排查

### Q1：上传模板后提示「仅支持 .docx 模板文件」

**原因**：上传的文件不是 `.docx` 格式（可能是 `.doc`）。
**解决**：用 Word 打开文件，另存为 `.docx` 格式后重新上传。

### Q2：上传后书签数量为 0

**原因**：模板中没有插入 Word 书签。
**解决**：按照本文第三章的方法，在 Word 中插入书签后重新上传。可使用 `insert_bookmarks.py` 脚本自动插入。

### Q3：上传时报「创建失败，请检查模板文件后重试」

**原因**：Python 微服务未启动，或模板文件损坏。
**解决**：
1. 确认 Python 服务已启动：访问 http://localhost:5000/api/health。
2. 尝试用 Word 打开模板文件，确认文件未损坏。
3. 查看控制台日志获取详细错误信息。

### Q4：导出的 Word 文件名为「未命名申报表.docx」

**原因**：学生填写的表单数据中，没有键名包含 `姓名`、`name`、`xm` 的字段。
**解决**：检查模板书签命名，确保至少有一个书签的名称包含上述关键词之一。

### Q5：导出的 Word 中某些字段为空

**原因**：学生未填写该字段，或书签名称与表单字段不匹配。
**解决**：
1. 在提交列表查看学生填写的数据。
2. 确认模板中的书签名称与表单提交的字段名一致。

### Q6：批量导出 ZIP 中有 `_导出失败.txt` 文件

**原因**：该学生的 Word 导出过程中出错（通常是 Python 服务异常或模板文件问题）。
**解决**：
1. 打开 txt 文件查看具体错误原因。
2. 尝试单独导出该学生的 Word，获取更详细的错误信息。
3. 检查 Python 服务日志。

### Q7：学生无法登录，提示「该活动已截止报名」

**原因**：活动已被教师截止，或已过截止时间。
**解决**：如需重新开放，教师可在活动列表重新创建活动（已截止的活动无法重新开启）。

### Q8：使用 `insert_bookmarks.py` 脚本报错「无法转换 .doc 文件」

**原因**：未安装 `pywin32`，或系统未安装 Microsoft Word。
**解决**：
1. 安装 pywin32：`pip install pywin32`。
2. 确认系统已安装 Microsoft Word（脚本依赖 Word COM 组件）。
3. 或者手动在 Word 中将 `.doc` 另存为 `.docx`，然后修改脚本跳过转换步骤。

### Q9：学生端没有显示证件照上传区域

**原因**：模板中没有包含 `photo` 或 `照片` 书签。
**解决**：
1. 在 Word 模板中照片所在单元格插入名为 `photo` 的书签。
2. 或运行 `insert_bookmarks.py` 脚本（已内置「照片」关键词识别，会自动插入 `photo` 书签）。
3. 重新上传模板创建活动。

### Q10：导出的 Word 中照片位置为空白

**原因**：学生提交时未上传证件照，或图片文件丢失。
**解决**：
1. 确认学生提交时选择了图片文件。
2. 检查服务器 `uploads/images/{活动ID}_{学号}/` 目录下是否存在图片文件。
3. 如果图片文件被误删，需学生重新提交（或教师重新创建活动让学生重填）。

### Q11：上传证件照时提示「图片大小不能超过 2MB」

**原因**：所选图片文件超过 2MB 限制。
**解决**：
1. 使用图片压缩工具（如微信截图、在线压缩网站）缩小图片。
2. 降低图片分辨率（建议 295×413 像素）。
3. 将图片另存为 JPG 格式（比 PNG 体积更小）。

## 九、技术架构简述（供技术人员参考）

```
[ 浏览器 ]
    |
    v
[ Java Spring Boot (8080) ]  --HTTP-->  [ Python Flask (5000) ]
    |                                         |
    +-- H2 数据库（开发）/ MySQL（生产）        +-- python-docx 处理 Word
    +-- Thymeleaf 渲染表单页面
    +-- JWT 鉴权（HttpOnly Cookie）
```

**关键接口**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/teacher/create` | POST | 上传模板，解析书签，暂存到 session |
| `/teacher/create/confirm` | GET/POST | 确认字段 / 保存活动 |
| `/teacher/export/{id}` | GET | 导出单个 Word |
| `/teacher/activity/{id}/export/zip` | GET | 批量导出 ZIP |
| `/api/parse-bookmarks` | POST | Python：解析书签 |
| `/api/fill-word` | POST | Python：填充 Word |

**数据流**：
1. 教师上传 `.docx` → Java 保存到 `uploads/` → 调用 Python `/api/parse-bookmarks` → 返回书签坐标 JSON。
2. 教师确认 → Java 将活动信息（含模板路径、书签 JSON）存入数据库。
3. 学生填写表单 → 文本数据以 JSON 存入 `submissions.form_data_json`；图片文件保存到 `uploads/images/{活动ID}_{学号}/`，路径存入 JSON。
4. 教师导出 → Java 读取模板路径和学生数据 → 调用 Python `/api/fill-word` → Python 检测图片路径并插入真实图片 → 返回填好的 `.docx` 字节流。
